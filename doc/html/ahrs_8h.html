float ahrs_att ( enum att_axis  dir)

Tells AHRS to return angle for a certain direction.

If the ahrs is in degrees mode the values will range per ahrs_range[dir].

The values returned will not changed until ahrs_att_update() is called and it returns true.

◆ ahrs_att_update()

bool ahrs_att_update ( )

Updates AHRS data.

Updates the values returned by ahrs_att to the newest complete set of data that has been received from the ahrs before some point in time within this function's lifetime.

Should only be called between complete "uses" of the attitude data to avoid using disparate data together (ie between runs of a PID routine).

Returns
True when there has been a new complete set of data received from the ahrs since the last time ahrs_att_update() has been called.

◆ ahrs_headingstatus()

uint_fast8_t ahrs_headingstatus ( )

Tells AHRS to return accuracy of current data.

Values: 1: uncertainty less than 2 degrees 2: uncertainty between 2 and 10 degres 3: uncertainty greater than 10 degrees However, it may give other values (I have known it to give the value 0).

Returns
The kHeadingStatus component from the ahrs associated with the current attitude data.

◆ ahrs_parse_att_reset()

void ahrs_parse_att_reset ( )

Discards incomplete AHRS data.

Causes any data from a current incomplete datagram to be discarded. The next received byte will be treated as potentially the start of a datagram.

Can be used due for, eg, a serial timeout for intra-datagram data.
